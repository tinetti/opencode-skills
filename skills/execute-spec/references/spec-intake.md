# Spec Intake

Read this before starting implementation.

## Confirm The Input

- What exact spec file or pasted artifact are you executing?
- What outcome does it require?
- What is explicitly in scope and out of scope?
- What files, systems, or commands does it name?
- What validations does it require?

## Classify Gaps

- Blocker: missing decision, contradictory requirements, or unknown target behavior.
- Assumption: low-risk detail you can state explicitly and proceed with.
- Drift: the spec no longer matches the current codebase.

## Default Behavior

- Proceed when the path is clear enough to make reversible progress.
- Ask only when the missing information changes behavior or scope.
- Keep notes on assumptions so they appear in the final handoff.
