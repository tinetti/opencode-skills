# Blocker Triage

Use this when execution stalls.

## Resolve It Yourself First

- Re-read the spec section that governs the behavior.
- Inspect the existing code and tests for implied intent.
- Check whether the blocker is really a local implementation bug.

## Ask The User Only When Needed

- The spec is contradictory.
- The decision changes user-visible behavior.
- The change would widen scope beyond the written spec.
- Required infrastructure or secrets are unavailable.

## How To Ask

- Ask one focused question.
- Include the recommended answer.
- State the consequence of each option.
